# 粒子特效切换 · 搜索过渡 · 首屏滚动吸附

## Context（为什么改）

当前状态（上一轮已完成）：
- `effects/warpParticles.js` + `styles/warpParticles.css` 已创建，并被 6 个 HTML 页面引用。
- 跃迁粒子是「开关式」：默认关闭，按钮浮动在视口右下角（`position:fixed`），画布覆盖整个视口。

用户新一轮反馈提出三点：
1. 粒子特效应**只在首页第一屏**出现；切换按钮的语义是**在「原 Three.js 粒子网络」与「新跃迁粒子」之间互切**，而非新特效的开关。
2. 导航栏搜索点击展开/关闭的**过渡不够流畅**。
3. **取消首页自由滑动**：向下滑动时直接自动过渡到第二屏。

本计划在不重构现有业务逻辑的前提下，做增量调整。

---

## 决策（已与用户确认）

- 滚动过渡：**JS 滚轮接管**（wheel/touch/键盘），非 CSS scroll-snap。
- 默认特效：**原 Three.js 粒子网络**（向后兼容，老访客无感知）。
- 切换按钮：**移入 Hero 内部**（不再浮动在视口右下角），随首屏一起滚走。

---

## 改动一：粒子特效互切 + 仅首页第一屏

### 1.1 从 5 个非首页 HTML 移除 warp 引用 `[MODIFIED]`

文件：`research.html`、`notes.html`、`share.html`、`notes/article.html`、`research/project.html`

每个文件删除两行（上一轮新增的）：
- `<link rel="stylesheet" href="(../)styles/warpParticles.css" />` 及其注释
- `<script src="(../)effects/warpParticles.js"></script>` 及其注释

`index.html` 保留这两行不动。

**原因**：跃迁特效仅首页第一屏需要；其他页面无 `.hero`，加载也无意义。

### 1.2 重写 `effects/warpParticles.js` `[MODIFIED]`（整文件重写，仍为自包含 IIFE）

核心变化：
- 目标容器从 `document.body` 改为 `.hero`；找不到 `.hero` 则直接 return（双保险）。
- 画布 `position:absolute` 贴合 `.hero`，与 `#particles-canvas` 同层（z-index:1）。
- **模式状态机**：`mode = 'network' | 'warp'`，默认 `'network'`，持久化键 `rs-warp`。
  - `network`：`.hero` 不加 `is-warp` 类 → CSS 显示 `#particles-canvas`、隐藏 `.warp-canvas`；warp raf 停止。
  - `warp`：`.hero` 加 `is-warp` 类 → CSS 隐藏 `#particles-canvas`（opacity:0）、显示 `.warp-canvas`；warp raf 启动。
- 切换按钮也 append 到 `.hero`（不再是 body），`position:absolute; bottom:36px; right:32px; z-index:3`，与 `.hero__scroll`（bottom:36px 居中）同高对称。
- 按钮图标两态：network（点阵网络图）/ warp（星芒放射图），点击翻转模式 + 持久化。
- `IntersectionObserver` 观察 `.hero`：warp 模式下离开视口暂停 raf，进入恢复（与 `particles.js` 既有逻辑对齐）。
- `resize()` 改用 `canvas.clientWidth/clientHeight`（贴 hero 尺寸），保留 Retina `dpr` 适配。
- 主题跟随：保留 `MutationObserver` 监听 `data-theme` 重读 `--fg`。
- `prefers-reduced-motion`：隐藏切换按钮、不启动 warp（与原行为一致；网络特效的温和运动由 `particles.js` 既有逻辑负责）。
- 搜索面板打开时禁用滚轮接管由 `home.js` 负责（见改动三），与本文件无关。

**对 `particles.js` 零改动**：Three.js 在 `warp` 模式下仍运行但 `opacity:0` 不可见（80 粒子开销可忽略）；其既有 `IntersectionObserver` 在离开首屏时本就会暂停。这样避免侵入 `particles.js`。

### 1.3 重写 `styles/warpParticles.css` `[MODIFIED]`

- `.warp-canvas`：`position:absolute; top:0; left:0; width:100%; height:100%; z-index:1; pointer-events:none; opacity:0; transition:opacity 0.8s var(--ease,ease);`
- `.hero.is-warp .warp-canvas { opacity:1; }`
- 新增（仅 index 加载，安全）：`#particles-canvas { transition: opacity 0.8s var(--ease,ease); }` `.hero.is-warp #particles-canvas { opacity:0; }`
- `.warp-toggle`：从 `position:fixed; z-index:1500` 改为 `position:absolute; bottom:36px; right:32px; z-index:3`（在 `.hero` 内）；其余视觉样式（边框/主题变量/blur）保留。
- **删除**原 `main,.page,.footer{position:relative;z-index:1}` 规则（画布不再覆盖整页，无需抬升主内容）。
- 保留 `prefers-reduced-motion` 隐藏规则。

## 改动二：搜索过渡更流畅

### 2.1 `css/components.css` `[MODIFIED]` —— `.search__results`

把 `display:none ⇄ block` 改为 opacity + transform + visibility 过渡，并加错峰时序：

```css
.search__results {
  /* 原有布局属性保留 */
  opacity: 0;
  transform: translateY(-8px);
  visibility: hidden;
  transition: opacity 0.3s var(--ease), transform 0.3s var(--ease), visibility 0s linear 0.3s;
}
.search.is-open .search__results {
  opacity: 1;
  transform: none;
  visibility: visible;
  transition: opacity 0.3s var(--ease) 0.15s, transform 0.3s var(--ease) 0.15s, visibility 0s;
}
```

- 开启时结果面板比顶栏晚 0.15s 淡入 + 下滑，层次更合理。
- 关闭时立即淡出（无延迟），与顶栏收起同步。

### 2.2 `js/search.js` `[MODIFIED]` —— `close()`

延迟清空结果，让面板有时间淡出（而非瞬间清空）：

```js
close() {
  this.isOpen = false;
  this.el.classList.remove('is-open');
  document.body.style.overflow = '';
  if (this.input) this.input.value = '';
  setTimeout(() => { if (!this.isOpen && this.results) this.results.innerHTML = ''; }, 320);
}
```

`open()` 的 `setTimeout(focus, 280)` 保持不变（与 0.5s 输入框展开时序匹配）。

## 改动三：首页滚轮吸附（hero ↔ sections）

### 3.1 `js/home.js` `[MODIFIED]` —— 新增 `bindHeroScrollSnap()`，在 `init()` 调用

逻辑：
- 取 `.hero` 与 `#sections`；`prefers-reduced-motion` 命中则直接 return（保留自由滚动）。
- `sectionsTop = sections.offsetTop`（首屏高度，约 100vh）。
- `locked` 锁，触发后 900ms 内忽略后续事件，防止触控板惯性重复触发。
- **wheel**（`passive:false`）：
  - 搜索打开时（`window.RS_SEARCH.isOpen`）直接 return，不干扰。
  - `deltaY>8 && scrollY < sectionsTop-60` → `scrollTo({top:sectionsTop,behavior:'smooth'})` + lock。
  - `deltaY<-8 && scrollY < sectionsTop+60 && scrollY>30` → `scrollTo({top:0,behavior:'smooth'})` + lock。
  - 其余区间（sections 中部及以下）自由滚动。
- **touch**：`touchstart` 记 Y，`touchend` 算方向，阈值 50px，同样锁定。
- **键盘**：ArrowDown/PageDown 在首屏 → sections；ArrowUp/PageUp 在 sections 顶部 → hero。
- **`.hero__scroll` 锚点**：拦截 `click`，`preventDefault` 后用 `scrollTo({top:sectionsTop,behavior:'smooth'})` + lock，保证平滑一致（不依赖全局 `scroll-behavior`）。

**与现有逻辑兼容**：
- 导航栏 `is-scrolled`（scrollY>24 切换）无需改：吸附过程中会自然触发滚动事件，导航栏平滑变实。
- `.hero` 内 `overflow:hidden`、`.sections` 顶部 `.waves`（90px）充当导航栏遮挡缓冲，吸附落点 `scrollY=sectionsTop` 时标题不会被导航栏盖住。
- 仅 `index.html` 加载 `home.js`，逻辑天然只作用于首页。

   - 首页第一屏默认显示 Three.js 粒子网络；点 Hero 右下按钮 → 平滑淡入跃迁粒子、网络淡出；再点 → 反向。刷新保持选择。

3. 控制台无 JS 错误。
